## Section 1 — UML Component Diagram Concepts

### Q1. A UML component diagram is a type of:
A. Structural UML diagram  
B. Mathematical proof only  
C. Natural language essay  
D. Behavioral-only diagram  

**Answer: A**  
**Justification:** Component diagrams show the structural organization of system components.

### Q2. A component diagram shows:
A. Only user emotions  
B. Only class method bodies  
C. How components are arranged and related  
D. Only CPU voltage  

**Answer: C**  
**Justification:** It represents components, dependencies, and interactions.

### Q3. System components are modular units that:
A. Cannot interact with anything  
B. Offer interfaces and encapsulate implementation  
C. Must expose all internal code  
D. Cannot be reused  

**Answer: B**  
**Justification:** Components encapsulate implementation and expose interfaces.

### Q4. Component diagrams help promote:
A. No architectural documentation  
B. Hidden dependencies only  
C. Random coupling  
D. Modularity and understanding of system architecture  

**Answer: D**  
**Justification:** Component diagrams clarify modular architecture.

### Q5. In a component diagram, a component may represent:
A. Only a user role  
B. Only a keyboard  
C. A software class, collection of classes, or subsystem  
D. Only a printed page  

**Answer: C**  
**Justification:** Components can represent modular software parts at different granularities.

### Q6. The standard symbol for a component is usually:
A. A rectangle with «component» stereotype  
B. A diamond only  
C. A stick figure only  
D. A cloud only  

**Answer: A**  
**Justification:** UML commonly shows components as rectangles labeled with the component stereotype.

### Q7. A component’s main function is to:
A. Store only exam marks  
B. Hide all possible communication  
C. Replace all interfaces  
D. Define and encapsulate functionality  

**Answer: D**  
**Justification:** Components package functionality for modularity and reuse.

### Q8. Interfaces in a component diagram specify:
A. Processor clock speed only

D. User passwords  

**Answer: B**  
**Justification:** Interfaces define communication contracts.

### Q9. A provided interface is commonly represented by:
A. A socket symbol  
B. A 3D box  
C. A lollipop symbol  
D. A dashed rectangle only

**Justification:** Provided interfaces are shown using circles/lollipops.

### Q10. A required interface is commonly represented by:
A. A stick figure  
B. A socket or half-circle  
C. A lollipop only  
D. A database cylinder only

**Justification:** Required interfaces are shown as sockets/half-circles.

### Q11. Interfaces help ensure that components can be:

### Q13. A solid line usually indicates:

### Q14. An assembly connector connects:

### Q15. Relationships in component diagrams help visualize:

## Section 10 — Scenario-Based MCQs

### Q121. A team wants to show how the Payment, Order, and Inventory modules depend on each other. Which diagram is best?
A. Deployment diagram  
B. Timing diagram  
C. Component diagram  
D. Profile diagram  

**Answer: C**  
**Justification:** Component diagrams show software modules and dependencies.

### Q122. A team wants to show that the web app runs on a browser, business logic runs on an application server, and data runs on a database server. Which diagram is best?
A. Component diagram only  
B. Deployment diagram  
C. Composite structure only  
D. Profile diagram  

**Answer: B**  
**Justification:** Deployment diagrams show software mapped to nodes.

### Q123. A component provides an AuthenticationService interface. Which symbol is most appropriate?
A. Lollipop  
B. Socket  
C. Timing line only  
D. 3D node  

**Answer: A**  
**Justification:** Provided interfaces are shown using lollipop notation.

### Q124. A component needs a PaymentGateway interface from another component. Which symbol is most appropriate?
A. 3D node  
B. Artifact rectangle only  
C. Lollipop  
D. Socket  

**Answer: D**  
**Justification:** Required interfaces are represented by socket notation.

### Q125. A Customer component uses an external Account component. Which relationship best shows reliance?
A. Actor generalization  
B. Dependency  
C. Deployment specification only  
D. Timing state  

**Answer: B**  
**Justification:** Dependency shows that one component relies on another.

### Q126. A request entering a component is forwarded to an internal subcomponent. Which notation fits?
A. «node»  
B. «profile»  
C. «artifact»  
D. «delegate»  

**Answer: D**  
**Justification:** Delegation forwards external interface requests internally.

### Q127. A Java application is packaged as a .jar file. In UML this file is best modeled as:
A. Node  
B. Required interface  
C. Artifact  
D. Communication path  

**Answer: C**  
**Justification:** JAR files are deployable physical software artifacts.

### Q128. A Docker container is used to package and deploy an application. This is most related to:
A. Containerization deployment pattern  
B. Profile diagram only  
C. Timing diagram  
D. Class inheritance  

**Answer: A**  
**Justification:** Docker is associated with containerization.

### Q129. A university system has separate presentation, application, and database tiers. This is an example of:
A. Composite structure only  
B. Profile customization  
C. Three-tier architecture  
D. Timing diagram  

**Answer: C**  
**Justification:** Three-tier architecture separates presentation, logic, and data.

### Q130. A hospital system has independently deployable patient, billing, and appointment services. This is closest to:
A. Single monolithic artifact only  
B. Microservices architecture  
C. UML profile diagram  
D. Timing-only model  

**Answer: B**  
**Justification:** Microservices organize functionality as independently deployable services.

### Q131. A deployment diagram for a banking system should include:
A. Client device, application server, database server, and communication paths  
B. Only class method bodies  
C. Only a lollipop interface  
D. Only object states over time  

**Answer: A**  
**Justification:** Deployment diagrams show devices, servers, and connections.

### Q132. A component diagram for a banking system should include:
A. Only network cables  
B. Only CPU and router hardware  
C. Only memory specifications  
D. Components such as Account, Transaction, Authentication, and their interfaces  

**Answer: D**  
**Justification:** Component diagrams model software modules and interfaces.

### Q133. A system’s components are too tightly coupled. Which diagram can help reveal dependency problems?
A. Profile diagram only  
B. Component diagram  
C. Use-case narrative only  
D. Timing diagram only  

**Answer: B**  
**Justification:** Component diagrams show dependency paths between modules.

### Q134. A system has poor performance because all requests go through one overloaded server. Which diagram helps show this bottleneck?
A. Composite diagram only  
B. Profile diagram only  
C. Deployment diagram  
D. Class comment table only  

**Answer: C**  
**Justification:** Deployment diagrams show nodes and traffic/communication paths that can reveal bottlenecks.

### Q135. A team wants to model a component’s exact boundary points for communication. Which element should be used?
A. Nodes only  
B. Deployment specification only  
C. Timing constraints only  
D. Ports  

**Answer: D**  
**Justification:** Ports define interaction points on component boundaries.

### Q136. A team wants to model the physical database file deployed on a server. Which element should be used?
A. Artifact  
B. Interaction overview  
C. Provided interface  
D. Required interface  

**Answer: A**  
**Justification:** Files/databases are artifacts in deployment/component contexts.

### Q137. A server has nested virtual machines. Which UML concept can represent hierarchical computing resources?
A. Lollipops only  
B. Required interfaces only  
C. Profile tags only  
D. Nested nodes  

**Answer: D**  
**Justification:** Nodes can include nested nodes to represent hierarchy.

### Q138. A diagram shows a browser node connected to a web server node over HTTPS. The line is a:
A. Artifact  
B. Communication path  
C. Required interface  
D. Component port  

**Answer: B**  
**Justification:** Communication paths show node-to-node communication channels/protocols.

### Q139. A component diagram is poor if it:
A. Shows components without interfaces or dependencies when these are important  
B. Shows relationships between components  
C. Shows modular units  
D. Uses any rectangles  

**Answer: A**  
**Justification:** Interfaces and dependencies are central information in component diagrams.

### Q140. A deployment diagram is poor if it:
A. Labels servers clearly  
B. Shows nodes  
C. Does not show where software components/artifacts are deployed  
D. Shows communication paths  

**Answer: C**  
**Justification:** Deployment diagrams must map software to hardware/virtual nodes.

### Q141. Which pair is correctly matched?

**Answer: B**  
**Justification:** The lecture maps development view to component diagrams.

### Q142. Which pair is correctly matched?
A. Physical view — Deployment diagram  
B. Process view — Profile diagram  
C. Logical view — Deployment diagram  
D. Development view — Timing diagram  

**Answer: A**  
**Justification:** Physical view corresponds to deployment diagrams.

### Q143. Which pair is incorrectly matched?
A. Artifact — physical file  
B. Node — execution environment  
C. Provided interface — socket symbol  
D. Required interface — socket symbol  

**Answer: C**  
**Justification:** Provided interfaces are represented by lollipops, not sockets.

### Q144. Which pair is incorrectly matched?
A. Port — boundary interaction point  
B. Node — physical or virtual execution environment  
C. Artifact — physical file  
D. Dependency — solid association line only  

**Answer: D**  
**Justification:** Dependency is commonly shown using a dashed arrow.

### Q145. Which statement best describes an assembly connector?
A. It customizes UML stereotypes  
B. It represents a physical server only  
C. It packages Java code into a JAR  
D. It connects a required interface to a provided interface  

**Answer: D**  
**Justification:** Assembly connectors connect required and provided interfaces.

### Q146. Which statement best describes a node?
A. A method body  
B. A UML stereotype only  
C. A physical or virtual place where software executes  
D. A provided interface only  

**Answer: C**  
**Justification:** Nodes represent execution resources.

### Q147. Which statement best describes an artifact?
A. A physical piece of software/data such as an executable or file  
B. A user role  
C. A run-time process only  
D. A communication protocol only  

**Answer: A**  
**Justification:** Artifacts are physical items used or produced by software.

### Q148. Which statement best describes a component?
A. A timing mark  
B. A modular software unit that encapsulates functionality  
C. A physical router only  
D. A user actor  

**Answer: B**  
**Justification:** Components are modular units with encapsulated implementation.

### Q149. Which statement best describes an interface?
A. A database backup  
B. A physical CPU  
C. A project schedule  
D. A contract specifying operations a component offers or requires  

**Answer: D**  
**Justification:** Interfaces define interaction contracts.

### Q150. Which statement best describes a port?
A. A precise interaction point on a component boundary  
B. A source-code file  
C. A physical server  
D. A UML profile extension  

**Answer: A**  
**Justification:** Ports specify where interfaces are exposed or required.

### Q151. Which statement about component diagrams is correct?
A. They cannot show interfaces  
B. They are not UML diagrams  
C. They help show dependencies and interactions among components  
D. They only show physical servers  

**Answer: C**  
**Justification:** Component diagrams represent software components and their relations.

### Q152. Which statement about deployment diagrams is correct?
A. They are only behavioral diagrams  
B. They show physical deployment of software on nodes  
C. They cannot include artifacts  
D. They only show inheritance  

**Answer: B**  
**Justification:** Deployment diagrams show components/artifacts deployed on hardware/virtual nodes.

### Q153. Which statement about informal block diagrams is correct?
A. They are useful for communication but may lack semantics  
B. They are useless for planning  
C. They are always mathematically precise  
D. They always show all externally visible properties  

**Answer: A**  
**Justification:** Block diagrams are easy to understand but often imprecise.

### Q154. Which statement about formal languages is correct?
A. They are always less precise than informal diagrams  
B. They provide mathematically precise semantics  
C. They cannot support verification  
D. They are identical to freehand sketches  

**Answer: B**  
**Justification:** Formal languages support unambiguous modeling and verification.

### Q155. Which statement about UML as semi-formal notation is correct?
A. It is only used for database backup  
B. It is always executable  
C. It uses standardized visual notation but may allow ambiguity  
D. It is completely informal with no standard symbols  

**Answer: C**  
**Justification:** UML is standardized but not always mathematically strict.

### Q156. Which choice best fits “communication between two deployment nodes”?
A. Component port only  
B. Artifact stereotype  
C. Required interface  
D. Communication path  

**Answer: D**  
**Justification:** Communication paths model node-to-node communication links.

### Q157. Which choice best fits “a service a component offers”?
A. Timing state  
B. Provided interface  
C. Node  
D. Deployment specification  

**Answer: B**  
**Justification:** Provided interfaces describe offered services.

### Q158. Which choice best fits “a service a component needs”?
A. Profile diagram  
B. Deployment node  
C. Artifact  
D. Required interface  

**Answer: D**  
**Justification:** Required interfaces describe needed services.

### Q159. Which choice best fits “hardware/software setup details”?
A. Deployment specification  
B. Lollipop notation  
C. Required interface  
D. Provided interface  

**Answer: A**  
**Justification:** Deployment specifications document configuration and setup characteristics.

### Q160. Which choice best fits “software packaged for deployment”?
A. Use case  
B. Actor  
C. Artifact  
D. Node  

**Answer: C**  
**Justification:** Artifacts include executables, libraries, scripts, and deployable files.
